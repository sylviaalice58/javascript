1️⃣ Arrow Functions in JavaScript

Arrow functions were introduced in ES6 (ECMAScript 2015) to make writing functions shorter and cleaner.

✅ Basic Syntax
Normal Function
function greet(name) {
    return "Hello " + name;
}

Arrow Function
const greet = (name) => {
    return "Hello " + name;
}
Even Shorter (Implicit Return)

If there is only one line:

const greet = name => "Hello " + name;
✅ Arrow Function with No Parameters
const sayHi = () => {
    console.log("Hi!");
}
✅ Arrow Function with Multiple Parameters
const add = (a, b) => a + b;

console.log(add(5, 3));  // 8
⭐ Special Characteristics of Arrow Functions
1️⃣ Shorter Syntax

They reduce code length.

2️⃣ No this Binding

Arrow functions do NOT have their own this.
They inherit this from their surrounding scope.

Example:

const person = {
    name: "Sylvia",
    greet: function() {
        setTimeout(() => {
            console.log("Hello " + this.name);
        }, 1000);
    }
};

person.greet();

Here, the arrow function keeps the correct this.

❌ When NOT to Use Arrow Functions

In object methods (sometimes)

As constructors (you cannot use new with arrow functions)

When you need your own this

2️⃣ Functions with Parameters in JavaScript

A parameter is a variable inside the function definition.
An argument is the actual value passed when calling the function.

✅ Example
function multiply(a, b) {   // a and b are parameters
    return a * b;
}

console.log(multiply(4, 5));  // 4 and 5 are arguments
✅ Default Parameters
function greet(name = "Guest") {
    return "Hello " + name;
}

console.log(greet());        // Hello Guest
console.log(greet("Alice")); // Hello Alice
✅ Rest Parameters

Used when you don’t know how many arguments will be passed.

function sum(...numbers) {
    let total = 0;
    for (let num of numbers) {
        total += num;
    }
    return total;
}

console.log(sum(1, 2, 3, 4));  // 10
3️⃣ Anonymous Functions in JavaScript

An anonymous function is a function without a name.

✅ Example
const greet = function(name) {
    return "Hello " + name;
};

This function has no name, but it is stored inside the variable greet.

✅ Anonymous Function as Callback

Very common in JavaScript:

setTimeout(function() {
    console.log("This runs after 2 seconds");
}, 2000);

✅ Anonymous Arrow Function
setTimeout(() => {
    console.log("Arrow function example");
}, 2000);







📦 Modules in JavaScript

A module in JavaScript is simply a file that contains code (variables, functions, classes) that can be exported and imported into another file.

Modules help you:

Organize code

Avoid global variables

Reuse code

Make large projects manageable

Since you're learning JS step by step, this is a very important concept when building real projects.

1️⃣ Why Modules Are Important

Before modules:

// Everything is global 😬
let name = "Sylvia";
function greet() {
    console.log("Hello " + name);
}

If your project grows, global variables can clash.

Modules solve this by keeping code private by default.

2️⃣ ES6 Modules (Modern JavaScript Way)

JavaScript uses:

export → to share code

import → to use shared code

🧩 Example 1: Named Export
📁 math.js
export function add(a, b) {
    return a + b;
}

export function multiply(a, b) {
    return a * b;
}
📁 main.js
import { add, multiply } from './math.js';

console.log(add(5, 3));       // 8
console.log(multiply(5, 3));  // 15

✔ Notice the { } — used for named exports.

🧩 Example 2: Default Export
📁 greet.js
export default function greet(name) {
    return "Hello " + name;
}
📁 main.js
import greet from './greet.js';

console.log(greet("Sylvia"));

✔ No { } when importing default export.

3️⃣ Using Modules in HTML

When working in the browser:

<script type="module" src="main.js"></script>

⚠ Important: Must use type="module".

4️⃣ CommonJS (Older Node.js Style)

Used mostly in Node.js before ES6 modules:

// Export
module.exports = add;

// Import
const add = require('./math');

Modern projects now prefer ES6 modules.

5️⃣ Real-Life Example

Imagine you are building:

payment.js → handles payments

auth.js → handles login/signup

database.js → handles DB connection

app.js → main file

Each file is a module.

This is very useful especially since you've been working with Flask APIs and backend logic — JavaScript backend (Node.js) works the same modular way.
